Проект "Автоматическое развертывание конфигураций платформы 1С"

Типовой сценарий развертывания:
- ставится запрет на работу с 1С, 
- выгоняются пользователи, 
- завершаются все регламентные задания, 
- далее идет обновление на последнюю версию из хранилища, 
- далее обновление базы данных из подключенного хранилища 1С,
- далее при необходимости выгрузка архивной копии (7 последних копий хранится, более ранние удаляются)
- затем запуск сеанса 1С:Предприятия для выполнения регламентных операций обновления в режиме Предприятия. После завершения обновления сеанс 1С:Предприятие завершает свою работу.
- после обновления включается разрешение на работу пользователей и запуск регламентных заданий.

Все шаги/параметры настраиваются хранятся в специальных ini-файлах

Подготовлено полностью автоматическое обновление любых информационных баз 1С (по команде или в планировщие)

В папке \\WorkServer\Share\Admin1C\

Тестовая Для Анализа – запускать \\WorkServer\Share\Admin1C\Тестовая_Ayun_Обновить.cmd
Тестовая БИЗНЕСА – запускать \\WorkServer\Share\Admin1C\ТестоваяБизнес_Обновить_НеТерминал.cmd
Тестовая ЭТАЛОН – запускать \\WorkServer\Share\Admin1C\ТестоваяОбщая_Обновить_НеТерминал.cmd

В наших базах (Анализ и Эталон) ожидание выхода пользователей не происходит, т.е. выход пользователей проходит очень быстро.
Полный цикл обновления занимает примерно 7-10 минут.

В базе ТЕСТ Бизнеса ожидание выхода пользователей 5 минуты.
Соответственно Полный цикл обновления занимает примерно 12-17 минут.

Схема обновления – ставится запрет на работу с 1С, выгоняются пользователи, завершаются все регламентные задания, 
далее идет обновление на последнюю версию из хранилища, далее обновление базы данных,
далее выгрузка архивной копии (7 последних копий хранится, более ранние удаляются)
после выгрузки включается разрешение на работу пользователей и запуск регламентных заданий.

Перед обновлением ИБ из указанных ИБ нужно выйти из Конфигуратора этой ИБ, если он запущен. Что на своей машине, что на чужой он не должен быть запущен!
Иначе будут проблемы работы с хранилищем.
Хранилище будет считать, что вход в хранилище для этой базы/логина выполнен, и не даст выполнить обновление.

При обновлении тестовой ЭТАЛОН происходит полный перегруз базы из эталона!
В остальных базах только обновление.

Запускать просто (для тех, кто не в курсе) – нажимаем сочетание клавиш Windows+R, вставляем указанный полный путь к файлу и нажать «Выполнить»

**Всем советую использовать именно такой режим обновления.**
**Минимум работы, все автоматом, плюс делаются бэкапы!**
